1-2. 반대 논리 및 대응
  • 딜러사 직접 경쟁·ROI 없음 — 알림톡 미집행 채널 94.5%(비즈채널 기준 87.6%). 딜러사가 못 닿는 미개척 시장이 타겟. 직접매출(연 4~18억) 외 CRM·채널 활성화가 진짜 목적.
  • 딜러사로 충분·템플릿 기능 부족 — 뿌리오 진입 허들을 못 넘는 소상공인도 있음. 첫 진입점이므로 완벽 불필요. 대체발송 미지원은 의도적 포지셔닝.
  • 전화번호 DB 없음 — 라이트(전화번호 보유) + 플러스(채팅 기반, 전화번호 불필요) 이중 커버.

2-1. 알림톡 플러스란

1:1 채팅 흐름 안에서 유저의 액션이 트리거가 되어 알림톡이 발송되는 구조.
전화번호 없이 카카오 계정 기반 발송 가능. 라이트가 사장님이 엑셀파일 업로드 방식으로 "직접 보내는 것"이라면, 플러스는 "카카오 내 씬에서 자동으로 보내지는 것".

3. CRM 연결 — 장기자산
3-1. 왜 CRM이 필요한가

라이트·플러스가 따로 놀아서는 안됨.
각 거점에서 발송된 알림톡들을 CRM으로 묶어야 소상공인 고객 데이터가 카카오와 사장님의 자산이 됨!

파트너센터 > 고객 관리 탭 (현재 없음)

┌─────────────────────────────┐
│ 내 고객 목록 │
│ ──────────────────────────── │
│ 고객A 전화번호O 채팅X 쿠폰X │
│ 고객B 전화번호X 채팅O 쿠폰X │ ← 채팅에서 온 고객
│ 고객C 전화번호X 채팅X 쿠폰O │ ← 쿠폰으로 온 고객
│ ──────────────────────────── │
│ [알림톡 보내기] [쿠폰 발송] │
└─────────────────────────────┘

여기가 지금 없는 부분. 고객 A·B·C가 같은 사람일 수도 있는데 카카오가 모르고, 사장님도 누군지 관리 불가능

예를 들어, 라라네일 사업자(기존 채널 친구수 70명 가정)는 아래의 고객과 액션이 있었음.

1) 알림톡 라이트를 통해 전화번호 5명 업로드 발송- 7/7 예약 고객 5명에게 전화 번호 기반 단체 예약 알림톡 발송. (친구 2 / 비친구 3 혼재 가정)

2) 톡스토어 구매 - 이미 친구였던 A가 톡스토어에서 네일 제품 구매 → 주문완료(배송) 알림톡이 발송되었음

3) 유저 B가 1:1 채팅을 통해 무통장 입금 후 예약, 사장님이 1:1채팅에서 예약 알림톡을 발송함.

기존에 1:1채팅 이력이 없지만, 김*수의 경우 엑셀파일 업로드를 통해서 친구관계 성립, 예약 알림톡을 통해 거래 확인되어 1:1 선톡 가능해지는 구조.

3-2. 확인 필요 사항
전제 : SME 사장님들이 고객데이터를 보고, 재방문 마케팅 메시지를 보내는것에 관심이 있을까? 에 대한 확인

3kuid로 들어온 접점(구매·채팅·쿠폰 등 다양한 접점이)이 묶일 수 있는지
4"통합 마케팅 수신 동의"가 있고 어디까지 커버하는지 (1번 가능해도 법적으로 가능한지)
5사장님 선톡이 정책상 가능 여부 (법적, 기술적 검토 완료 가정) 제일 큰 리스크
3-3. 메시지 발송

알림톡+를 통해 쌓인 고객 데이터는 친구 전환 → 채널메시지 → CRM으로 이어지는 장기 퍼널의 시작점.
친구 전환이 있지 않더라도, 거래 내역 기반 비친구 메시지 활용 가능.
